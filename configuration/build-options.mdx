## Overview

The `build` section of your configuration controls how Electrobun bundles your application code, manages assets, and produces build artifacts.

## Bun Process Configuration

Configure the main Bun process that runs your application's backend logic:

<ParamField path="build.bun.entrypoint" type="string" default="src/bun/index.ts">
  Entry point for the main Bun process. This is the TypeScript/JavaScript file that serves as your application's backend entry point.
  
  ```typescript
  build: {
    bun: {
      entrypoint: "src/bun/index.ts",
    },
  }
  ```
</ParamField>

### Bun.build() Options

The `build.bun` section accepts all [Bun.build()](https://bun.sh/docs/bundler) options (plugins, sourcemap, minify, define, etc.) in addition to the entrypoint:

```typescript
build: {
  bun: {
    entrypoint: "src/bun/index.ts",
    minify: true,
    sourcemap: "external",
    define: {
      "process.env.NODE_ENV": '"production"',
    },
  },
}
```

## View Configuration

Configure browser views (renderer processes) for your application:

<ParamField path="build.views" type="object">
  Browser view build configurations. Each view is a key-value pair where the key is the view name and the value contains the entrypoint and optional Bun.build() options.
  
  Each view accepts all [Bun.build()](https://bun.sh/docs/bundler) options (plugins, sourcemap, minify, define, etc.) in addition to the entrypoint.
  
  ```typescript
  build: {
    views: {
      mainview: {
        entrypoint: "src/mainview/index.ts",
      },
      "test-runner": {
        entrypoint: "src/test-runner/index.ts",
        minify: true,
      },
      "playgrounds/file-dialog": {
        entrypoint: "src/playgrounds/file-dialog/index.ts",
        sourcemap: "inline",
      },
    },
  }
  ```
</ParamField>

<ParamField path="build.views[viewName].entrypoint" type="string" required>
  Entry point for this view's TypeScript/JavaScript code.
</ParamField>

## Asset Management

<ParamField path="build.copy" type="object">
  Files to copy directly to the build output. Key is source path, value is destination path relative to the build folder.
  
  Useful for HTML files, CSS, images, and other static assets that don't need bundling.
  
  ```typescript
  build: {
    copy: {
      "src/mainview/index.html": "views/mainview/index.html",
      "src/mainview/index.css": "views/mainview/index.css",
      "assets/electrobun-logo-32-template.png": "views/assets/electrobun-logo-32-template.png",
      "dist/assets": "views/mainview/assets", // Copy entire directory
    },
  }
  ```
</ParamField>

## Output Configuration

<ParamField path="build.buildFolder" type="string" default="build">
  Output folder for built application files.
  
  ```typescript
  build: {
    buildFolder: "build",
  }
  ```
</ParamField>

<ParamField path="build.artifactFolder" type="string" default="artifacts">
  Output folder for distribution artifacts (installers, packages, etc.).
  
  ```typescript
  build: {
    artifactFolder: "artifacts",
  }
  ```
</ParamField>

<ParamField path="build.targets" type="string">
  Build targets to compile for. Can be:
  - `"current"` - Only the current platform
  - `"all"` - All supported platforms
  - Comma-separated list like `"macos-arm64,win-x64,linux-x64"`
  
  Available targets:
  - `macos-arm64` - macOS Apple Silicon
  - `macos-x64` - macOS Intel
  - `win-x64` - Windows 64-bit
  - `linux-x64` - Linux 64-bit
  
  ```typescript
  build: {
    targets: "macos-arm64,win-x64",
  }
  ```
</ParamField>

## ASAR Packaging

<ParamField path="build.useAsar" type="boolean" default="false">
  Enable ASAR archive packaging for bundled assets. When enabled, all files in the Resources folder will be packed into an `app.asar` archive.
  
  Benefits:
  - Faster file access
  - Reduced file system overhead
  - Basic obfuscation of source code
  
  ```typescript
  build: {
    useAsar: true,
  }
  ```
</ParamField>

<ParamField path="build.asarUnpack" type="string[]" default='["*.node", "*.dll", "*.dylib", "*.so"]'>
  Glob patterns for files to exclude from ASAR packing (extract to `app.asar.unpacked`).
  
  Useful for:
  - Native modules that need to be loaded dynamically
  - Executables that need to be run directly
  - Files accessed by external processes
  
  ```typescript
  build: {
    useAsar: true,
    asarUnpack: [
      "*.node",
      "*.dll",
      "*.dylib",
      "*.so",
      "bin/**/*",
    ],
  }
  ```
</ParamField>

## Version Overrides

<ParamField path="build.cefVersion" type="string">
  Override the CEF (Chromium Embedded Framework) version.
  
  Format: `"CEF_VERSION+chromium-CHROMIUM_VERSION"`
  
  Example: `"144.0.11+ge135be2+chromium-144.0.7559.97"`
  
  Check the [electrobun-cef-compat](https://github.com/blackboardsh/electrobun-cef-compat) compatibility matrix for tested combinations before overriding. Using an untested version may cause runtime issues.
  
  ```typescript
  build: {
    cefVersion: "144.0.12+g1a1008c+chromium-144.0.7559.110",
  }
  ```
</ParamField>

<ParamField path="build.bunVersion" type="string">
  Override the Bun runtime version.
  
  Format: semver string (e.g., `"1.4.2"`)
  
  This downloads the specified Bun version from GitHub releases and uses it instead of the version bundled with this Electrobun release.
  
  ```typescript
  build: {
    bunVersion: "1.3.7",
  }
  ```
</ParamField>

## Complete Example

Here's a comprehensive build configuration example from the Kitchen Sink app:

```typescript
import type { ElectrobunConfig } from "electrobun";

export default {
  app: {
    name: "Electrobun Kitchen Sink",
    identifier: "sh.blackboard.electrobun-kitchen",
    version: "1.13.1",
  },
  build: {
    useAsar: true,
    bun: {
      entrypoint: "src/bun/index.ts",
    },
    views: {
      "test-runner": {
        entrypoint: "src/test-runner/index.ts",
        minify: true,
      },
      "test-harness": {
        entrypoint: "src/test-harness/index.ts",
      },
      "playgrounds/file-dialog": {
        entrypoint: "src/playgrounds/file-dialog/index.ts",
      },
    },
    copy: {
      "src/test-runner/index.html": "views/test-runner/index.html",
      "src/test-runner/index.css": "views/test-runner/index.css",
      "src/test-harness/index.html": "views/test-harness/index.html",
      "src/playgrounds/file-dialog/index.html": "views/playgrounds/file-dialog/index.html",
      "assets/electrobun-logo-32-template.png": "views/assets/electrobun-logo-32-template.png",
    },
    buildFolder: "build",
    artifactFolder: "artifacts",
  },
  scripts: {
    postBuild: "./buildScript.ts",
  },
} satisfies ElectrobunConfig;
```
